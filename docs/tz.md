# ТЕХНИЧЕСКОЕ ЗАДАНИЕ (ТЗ)
**Проект:** Private Photography Delivery Platform (Self-hosted Wfolio alternative)
**Заказчик:** Профессиональный фотограф
**Цель:** Создание платформы для сдачи фото-проектов клиентам (B2B) с фокусом на эстетику, удобство просмотра, защиту контента и простоту администрирования.

## 1. Глоссарий и Архитектура Сущностей
Система должна иметь строгую иерархию контента:
1.  **Project (Проект):** Верхний уровень сущности. Один Проект = Один Клиент/Съемка. Проект имеет глобальные настройки (пароль, срок жизни, разрешение на скачивание) и единую Hero-секцию.
2.  **Gallery (Галерея):** Подразделение внутри Проекта. Физически соответствует папке с файлами. На фронтенде отображается как отдельный таб (вкладка) внутри Проекта.
3.  **Photo (Фотография):** Атомарная единица контента, принадлежащая Галерее.

## 2. Административная Панель (Back-office)

### 2.1. Управление Проектами
*   **Создание:** Админ создает Проект, задавая его название.
*   **Глобальные настройки Проекта:**
    *   **Access Control:** Переключатель "Публичный" / "Защищен паролем" (с полем для ввода пароля).
    *   **Download Permissions:** Глобальный переключатель "Разрешить скачивание" (Вкл/Выкл).
    *   **Expiration Date (Срок жизни):** Обязательное поле с выбором даты и времени. По истечении срока Проект скрывается/архивируется.
*   **Статистика:** Дашборд проекта, показывающий количество уникальных просмотров и количество фактов скачивания (отдельных фото и ZIP-архивов).

### 2.2. Загрузка и Управление Медиа (Workflow)
*   **Drag-and-Drop Загрузка:** Админ перетаскивает папки с локального компьютера в зону загрузки.
*   **Автоматическое создание галерей:** Каждая загруженная папка автоматически конвертируется в отдельную Галерею (Таб) внутри текущего Проекта. Название папки = Название Таба.
*   **Сортировка:**
    *   По умолчанию: Алфавитная.
    *   Ручное управление: Drag-and-drop перетаскивание фотографий внутри галереи для изменения порядка.
*   **Редактирование на лету:** Добавление, удаление или замена фото без изменения URL-адреса Проекта/Галереи (permalinks сохраняются).

### 2.3. Обработка Изображений (Image Processing Pipeline)
*   **Web-Optimization:** При загрузке система генерирует сжатые веб-версии (WebP/JPEG) для быстрого отображения. Оригиналы сохраняются нетронутыми.
*   **Hero-Image Selection:** Система автоматически назначает *самую первую фотографию Проекта* (первое фото первой галереи) в качестве обложки (Hero-секции).

## 3. Публичный Интерфейс (Front-office / Client Area)

### 3.1. Дизайн и Брендирование
*   **White-Label подход:** Интерфейс не содержит брендинга фотографа. Допускается только кастомный Favicon и неброская ссылка в футере.
*   **Единый стиль:** Все проекты используют один CSS-дизайн. Кастомизация под клиента не предусмотрена.
*   **Язык интерфейса:** Строго Английский (UI elements, buttons, tooltips).
*   **Responsive Design:** Полная адаптивность (мобильные, планшеты, десктопы).

### 3.2. Структура Страницы Проекта
*   **Hero-Section:** Занимает 100% высоты первого экрана (`100vh`). Изображение масштабируется по принципу `cover`, заполняя весь экран.
*   **Навигация:** Под Hero-секцией расположены табы с названиями галерей.
*   **Индикатор срока жизни:** Рядом с кнопками скачивания отображается текст: *"Gallery available until: [Date]"*.

### 3.3. Отображение Галереи (Layout)
*   **Justified Grid:** Единый стиль для всех галерей.
    *   Фотографии выстраиваются в строки.
    *   Все фото в одной строке имеют **одинаковую высоту**.
    *   Ширина варьируется, сохраняя оригинальные пропорции (aspect ratio) без кроппинга.
    *   Строки плотно прилегают друг к другу.

### 3.4. Система Просмотра (Image Viewer / Lightbox)
*   **Активация:** Клик по фото открывает полноэкранный оверлей.
*   **Навигация:** Стрелки на экране, клавиатура (`Left`, `Right`, `Esc`), свайпы на мобильных.
*   **Зум:** Десктоп (колесико мыши/клик), Мобильные (pinch-to-zoom).
*   **Слайдшоу:** Не требуется.

### 3.5. Скачивание и Защита (Download & Security)
*   **Скачивание оригиналов:** Клиент скачивает только исходные файлы (без веб-сжатия).
*   **Одиночное скачивание:** Кнопка "Download" внутри Viewer'а.
*   **Массовое скачивание (ZIP):** Кнопка "Download All" генерирует ZIP-архив с оригиналами.
    *   *Требование к генерации:* Скорость генерации не является приоритетом, критически важно 100% сохранение оригинального качества и целостности файлов. Архив должен формироваться полностью на сервере (или в фоновом процессе / background job) до начала отдачи клиенту, чтобы избежать таймаутов и обрывов при скачивании больших объемов.
*   **Anti-Theft Protection:** Если "Разрешить скачивание" = `false`:
    *   Кнопки скачивания скрываются.
    *   Блокируется контекстное меню (правый клик) на изображениях.
    *   Блокируется нативный drag-and-drop изображений из браузера.

### 3.6. Социальные функции (Sharing)
*   **Share Project:** Кнопка "Share" на уровне Проекта. Генерирует прямую ссылку.
*   **Share Photo:** Кнопка "Share" внутри Viewer'а. Генерирует глубокую ссылку (deep link), запускающую Viewer на конкретном кадре.
    *   *Безопасность:* Если Проект защищен паролем, переход по глубокой ссылке требует предварительной аутентификации (ввода пароля от Проекта) перед показом фотографии.
